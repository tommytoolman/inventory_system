1. Overview

This integration connects RIFF Technology's multi-platform inventory management system with WooCommerce, enabling bidirectional synchronisation of product listings, inventory levels, and order notifications.

What This Integration Does

📤 Outbound Sync

Push products from RIFF to your WooCommerce store automatically.

📥 Inbound Sync

Import products from WooCommerce into RIFF's unified inventory.

🔄 Real-time Updates

Keep stock levels in sync across all platforms using webhooks.

📦 Order Notifications

Automatically update inventory when products sell on WooCommerce.

2. Prerequisites

Required Software

  • WordPress 4.4 or higher
  • WooCommerce 3.5 or higher (REST API v3 support)
  • SSL Certificate (HTTPS required for API access)
  • Pretty Permalinks enabled in WordPress

WordPress Configuration

  1. Navigate to Settings → Permalinks
  2. Select any option except "Plain" (recommended: "Post name")
  3. Click "Save Changes"
💡 Why Pretty Permalinks?

WooCommerce REST API endpoints use custom URLs like /wp-json/wc/v3/products. Plain permalinks don't support these custom routes, causing API requests to fail.

3. Setting Up WooCommerce

Enable REST API Access

WooCommerce REST API is enabled by default in WooCommerce 3.5+. To verify:

  1. Log into WordPress admin dashboard

5. Integration Architecture

System Overview

The integration uses WooCommerce REST API v3 for all operations:

RIFF Technology Platform
        ↕️
WooCommerce API Client (Python)
        ↕️
WooCommerce REST API v3
    (HTTPS/Basic Auth)
        ↕️
WordPress/WooCommerce Store

API Endpoints Used

Endpoint Method Purpose
/wp-json/wc/v3/products GET List all products
/wp-json/wc/v3/products/{id} GET Get single product
/wp-json/wc/v3/products POST Create new product
/wp-json/wc/v3/products/{id} PUT Update product
/wp-json/wc/v3/products/{id} DELETE Delete product
/wp-json/wc/v3/products/batch POST Bulk operations
/wp-json/wc/v3/orders GET Monitor orders
/wp-json/wc/v3/webhooks POST Create webhooks

Authentication Flow (Sandbox)

1. Client Request
   └─> Include Basic Auth header:
       Authorization: Basic base64(consumer_key:consumer_secret)

2. WooCommerce Verification
   └─> Validate credentials
   └─> Check user permissions
   
3. API Response
   └─> Return JSON data or error

6. Implementation Steps

Environment Variables

Store your credentials securely using environment variables:

# .env (NEVER commit this file!)
WC_STORE_URL=https://your-sandbox-store.com
WC_CONSUMER_KEY=ck_your_consumer_key_here
WC_CONSUMER_SECRET=cs_your_consumer_secret_here
WC_AUTH_METHOD=basic
WC_SANDBOX_MODE=true

Python Implementation Example

import os
import requests
from requests.auth import HTTPBasicAuth

class WooCommerceClient:
    def __init__(self):
        self.store_url = os.getenv('WC_STORE_URL')
        self.consumer_key = os.getenv('WC_CONSUMER_KEY')
        self.consumer_secret = os.getenv('WC_CONSUMER_SECRET')
        self.base_url = f'{self.store_url}/wp-json/wc/v3'
    
    def get_products(self, params=None):
        """Fetch products from WooCommerce"""
        url = f'{self.base_url}/products'
        response = requests.get(
            url,
            auth=HTTPBasicAuth(self.consumer_key, self.consumer_secret),
            params=params
        )
        response.raise_for_status()
        return response.json()
    
    def create_product(self, product_data):
        """Create a new product in WooCommerce"""
        url = f'{self.base_url}/products'
        response = requests.post(
            url,
            auth=HTTPBasicAuth(self.consumer_key, self.consumer_secret),
            json=product_data
        )
        response.raise_for_status()
        return response.json()
    
    def update_product(self, product_id, updates):
        """Update an existing product"""
        url = f'{self.base_url}/products/{product_id}'
        response = requests.put(
            url,
            auth=HTTPBasicAuth(self.consumer_key, self.consumer_secret),
            json=updates
        )
        response.raise_for_status()
        return response.json()

# Usage
client = WooCommerceClient()
products = client.get_products(params={'per_page': 10})

7. Synchronisation Strategy

Data Mapping

How RIFF fields map to WooCommerce:

RIFF Field WooCommerce Field Notes
product.id meta_data._riff_id Stored in custom meta field
product.title name Direct mapping
product.sku sku Must be unique
product.price regular_price String format required
product.quantity stock_quantity Integer value
product.description description Supports HTML
product.images[] images[].src Array of image URLs

Sync Approaches

Option 1: Webhook-Based (Recommended)

  • ✅ Real-time updates
  • ✅ Efficient (no polling)
  • ✅ Immediate inventory sync
  • ❌ Requires publicly accessible endpoint
// Create webhook in WooCommerce
POST /wp-json/wc/v3/webhooks
{
    "name": "RIFF Product Updated",
    "topic": "product.updated",
    "delivery_url": "https://riff-api.com/webhooks/woocommerce/product-updated",
    "secret": "your-webhook-secret-here"
}

Option 2: Polling-Based (Fallback)

  • ✅ Simpler implementation
  • ✅ Works with any network setup
  • ❌ Delayed updates (poll every 5-15 mins)
  • ❌ Higher API usage
# Poll for products modified since last sync
params = {
    'modified_after': last_sync_timestamp,
    'per_page': 100
}
products = client.get_products(params=params)

Conflict Resolution

When the same product is modified in both RIFF and WooCommerce:

  1. Compare timestamps (modified_date in WooCommerce, updated_at in RIFF)
  2. Most recent modification wins
  3. Log conflicts for manual review
  4. Option: Implement manual conflict resolution UI

9. Troubleshooting

Common Issues

❌ "Consumer key is missing" Error

Cause: Server cannot parse Authorization header

Solution: Use query string authentication instead:

url = f'{base_url}/products?consumer_key={key}&consumer_secret={secret}'
response = requests.get(url)  # No auth header needed

❌ 404 Not Found on API Endpoints

Cause: Permalinks not configured correctly

Solution:

  1. Go to Settings → Permalinks
  2. Select "Post name" or any option except "Plain"
  3. Click Save Changes
  4. Try API request again

❌ 401 Unauthorised Error

Causes & Solutions:

  • Wrong credentials: Regenerate API key in WooCommerce
  • User lacks permissions: Use administrator account
  • SSL issues: Ensure store has valid HTTPS certificate

❌ 429 Rate Limit Error

Solution: Implement exponential backoff:

import time

def make_request_with_retry(url, max_retries=3):
    for attempt in range(max_retries):
        response = requests.get(url, auth=auth)
        
        if response.status_code == 429:
            retry_after = int(response.headers.get('Retry-After', 60))
            print(f'Rate limited. Waiting {retry_after}s...')
            time.sleep(retry_after)
            continue
        
        return response
    
    raise Exception('Max retries exceeded')

❌ Duplicate SKU Error

Cause: SKU already exists in WooCommerce

Solutions:

  1. Check if product exists before creating
  2. Update existing product instead of creating new
  3. Use unique SKU prefix (e.g., "RIFF-" + original SKU)

Before deploying to production with real customer data, you MUST implement OAuth 1.0a authentication. Basic Auth is ONLY acceptable for sandbox/development environments.

Security Best Practices

  • ✅ Never commit credentials to version control
  • ✅ Use environment variables or secret managers (AWS Secrets Manager, Vault)
  • ✅ Rotate API credentials quarterly
  • ✅ Monitor API usage for unusual patterns
  • ✅ Implement request signing (OAuth 1.0a)
  • ✅ Verify webhook signatures
  • ✅ Use HTTPS for all API communication
  • ✅ Log all authentication failures
  • ✅ Implement rate limiting on your webhook endpoints

Monitoring & Alerts

Set up monitoring for:

  • API error rates (>5% should trigger alert)
  • Authentication failures
  • Sync failures or delays
  • Webhook delivery failures
  • Rate limit hits
  • Stock discrepancies between platforms
